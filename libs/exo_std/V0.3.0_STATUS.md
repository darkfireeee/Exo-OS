### 5. **TLS (Thread-Local Storage) Complet** ✅
**Fichier**: `src/thread/tls.rs` (~400 lignes)
**Status**: COMPLET

**Fonctionnalités implémentées**:
- ✅ `TlsTemplate` - Structure de template TLS depuis ELF
- ✅ `TlsBlock` - Bloc TLS par thread
- ✅ `allocate()` - Allocation et initialisation TLS
- ✅ `set_fs_base()` - Configuration registre FS via arch_prctl
- ✅ `get_fs_base()` / `set_gs_base()` - Gestion registres FS/GS
- ✅ `write_at()` / `read_at()` - Accès TLS typé
- ✅ Global TLS template management
- ✅ Integration syscall ArchPrctl (158)
- ✅ 6 tests unitaires

**Architecture**:
```rust
pub struct TlsTemplate {
    pub addr: usize,        // Adresse .tdata
    pub file_size: usize,   // Taille données initialisées
    pub mem_size: usize,    // Taille totale (.tdata + .tbss)
    pub align: usize,       // Alignement requis
}

pub struct TlsBlock {
    data: *mut u8,          // Données TLS allouées
    size: usize,            // Taille allocation
    align: usize,           // Alignement
}
```

**Process d'allocation**:
1. Allouer mémoire alignée
2. Copier .tdata (données initialisées)
3. Zero-init .tbss (données non-init)
4. arch_prctl(ARCH_SET_FS) pour configurer FS base

**API Complète**:
| Fonction | Description |
|----------|-------------|
| `TlsBlock::allocate()` | Alloue TLS depuis template |
| `set_fs_base()` | Configure FS register |
| `get_fs_base()` | Lit FS register |
| `set_gs_base()` | Configure GS register |
| `write_at<T>()` | Écrit valeur typée |
| `read_at<T>()` | Lit valeur typée |
| `init_tls_template()` | Init template global |
| `get_tls_template()` | Récupère template |
| `allocate_current_thread_tls()` | Alloue pour thread actuel |
| `setup_thread_tls()` | Helper pour thread::spawn() |

**Performance**:
- Allocation: O(n) où n = mem_size (copie + zero-init)
- Configuration FS: 1 syscall arch_prctl
- Accès TLS: %fs:offset = 1 cycle (direct CPU)

**Lignes Ajoutées**: ~400 lignes + 6 tests
