# CORR-71 — APIC timer DOWN-counter : deadline inversée (MIN-03 + MIN-10)

**Sévérité :** 🟡 MINEURE (CRITIQUE sur bare-metal : boucle ~5 secondes ou spin infini)  
**Fichier :** `kernel/src/exophoenix/forge.rs:184, 293–295`  
**Impact :** `checklist_tlb_shootdown()` et `drain_dma_queues()` tournent ~5s au lieu de 100µs

---

## Problème

```rust
// forge.rs:182–186 (première occurrence)
let deadline = stage0::ticks_per_us()
    .saturating_mul(100)
    .saturating_add(read_apic_ticks() as u64);
while (read_apic_ticks() as u64) < deadline {
    // APIC timer est DÉCROISSANT.
    // read_apic_ticks() part de start (ex: 300_000) et descend vers 0.
    // deadline = start + 100*ticks_per_us = ex: 400_000
    // Condition : current < 400_000
    // → current commence à 300_000 (< 400_000 : vrai) et descend vers 0.
    // La boucle s'exécute jusqu'à ce que l'APIC timer wrappe (plusieurs secondes).
    core::hint::spin_loop();
}

// forge.rs:293–295 (deuxième occurrence identique)
let deadline = (read_apic_ticks() as u64)
    .saturating_add(ticks_per_us.saturating_mul(100));
while (read_apic_ticks() as u64) < deadline {
    core::hint::spin_loop();
}
```

## Correction

Remplacer la logique basée sur le timer APIC (DOWN-counter, complexe) par `ktime_get_ns()` (monotonic, simple) :

```rust
// forge.rs — CORRECTION des deux occurrences

// Occurrence 1 : forge.rs:182–186
{
    use crate::arch::x86_64::time::ktime::{ktime_get_ns, ktime_deadline_us};
    
    // 100µs de délai.
    let deadline_ns = ktime_deadline_us(100);
    while ktime_get_ns() < deadline_ns {
        core::hint::spin_loop();
    }
}

// Occurrence 2 : forge.rs:293–295
{
    use crate::arch::x86_64::time::ktime::{ktime_get_ns, ktime_deadline_us};
    
    let deadline_ns = ktime_deadline_us(100);
    while ktime_get_ns() < deadline_ns {
        core::hint::spin_loop();
    }
}
```

### Si l'usage de ktime n'est pas possible (early context sans TSC calibré)

```rust
// Alternative : lire l'APIC timer correctement (DOWN-counter)
fn apic_wait_us(us: u64) {
    let ticks_per_us = stage0::ticks_per_us();
    let ticks_to_wait = ticks_per_us.saturating_mul(us);

    let start_ticks = read_apic_ticks() as u64;
    
    // DOWN-counter : attendre que (start - current) >= ticks_to_wait
    loop {
        let current = read_apic_ticks() as u64;
        // wrapping_sub gère le wrap-around du timer
        let elapsed = start_ticks.wrapping_sub(current);
        if elapsed >= ticks_to_wait {
            break;
        }
        core::hint::spin_loop();
    }
}

// Utilisation :
apic_wait_us(100); // 100µs — correct
```

## MAJ-08 — Commentaire offset TCB trompeur dans exocage.rs

```rust
// kernel/src/security/exocage.rs:17–21

// AVANT (trompeur) :
//   _cold_reserve[144]   shadow_stack_token : u64   (PKS domain TcbHot)
//   _cold_reserve[152]   cet_flags          : u8    (bit 0 = CET_EN)
//   _cold_reserve[153]   threat_score_u8    : u8    (0..=100)
//   _cold_reserve[160]   pt_buffer_phys     : u64   (Phase 4, LBR/PT futur)

// APRÈS (correct) :
// _cold_reserve est un tableau [u8; 88] commençant à l'offset TCB 144.
// Les notations ci-dessous donnent l'OFFSET ABSOLU DANS LE TCB,
// pas l'index dans _cold_reserve[].
//
// Offset TCB | Index cold_reserve | Champ                    | Type
// -----------|--------------------|--------------------------|------
// 144        | [0]                | shadow_stack_token       | u64
// 152        | [8]                | cet_flags                | u8    (bit 0 = CET_EN)
// 153        | [9]                | threat_score_u8          | u8    (0..=100)
// 160        | [16]               | pt_buffer_phys           | u64   (Phase 4)
// 168..232   | [24..88]           | réservé                  | [u8; 64]
//
// Pour accéder à shadow_stack_token : _cold_reserve[0] (offset dans tableau)
// Constante OFF_SHADOW_STACK_TOKEN = 0 est CORRECTE.
```

## CRIT-07 (nuancé) — A_LIVENESS_MIRROR_PHYS interface non définie

```rust
// kernel/src/exophoenix/sentinel.rs:40

// AVANT (arbitraire, non documenté) :
// const A_LIVENESS_MIRROR_PHYS: u64 = KERNEL_LOAD_PHYS_ADDR + 0x280;

/// Offset physique dans l'image Kernel A où B lit le nonce de liveness.
///
/// # Protocole d'acquittement (doit être respecté par les deux kernels)
///
/// Kernel A DOIT écrire à cet offset précis à intervalles réguliers

///   - Lit `PHYS_MAP_BASE + A_LIVENESS_MIRROR_PHYS` (ligne 221)
///   - Compare le nonce lu au nonce attendu (incrémentation périodique)
///
/// Côté Kernel A (doit implémenter) :
///   - Exporter la constante `EXOPHOENIX_LIVENESS_OFFSET = 0x280`
///   - Dans la boucle principale : écrire `liveness_nonce++` à cet offset
///
/// FIXME: Cette interface n'est pas encore définie formellement dans
```

## MIN-09 — POOL_R3_SIZE_BYTES = 0 si aucun device PCI détecté

```rust
// kernel/src/exophoenix/stage0.rs

// Dans la séquence d'init :
let pci_device_count = enumerate_pci_devices();
let pool_r3_size = POOL_R3_SIZE_BYTES.load(Ordering::Acquire);

// AJOUTER : guard si aucun device trouvé
let pool_r3_size = if pool_r3_size == 0 {
    // Aucun device PCI Ring1 détecté.
    // Utiliser une taille minimale pour permettre l'initialisation.
    // Les drivers Ring1 pourront toujours démarrer sans DMA.
    crate::memory::core::PAGE_SIZE as u64 * 16 // 64KB minimum
} else {
    pool_r3_size
};

let _ = init_pool_r3_from_stage0_size(pool_r3_size);
```
