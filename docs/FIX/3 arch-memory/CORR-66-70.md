# CORR-66 — HPET MMIO non mappé (MAJ-05)

**Sévérité :** 🟠 MAJEURE — Crash bare-metal au premier accès HPET  
**Fichier :** `kernel/src/arch/x86_64/acpi/hpet.rs:190`

---

## Problème

```rust
// hpet.rs:169–192
fn map_hpet_mmio_fixmap() {
    // TODO bare-metal : ajouter le remap 4K avec PAGE_FLAGS_MMIO dans le fixmap.
    //   use crate::arch::x86_64::paging::{map_4k_page, PAGE_FLAGS_MMIO};
    //   use crate::memory::core::layout::{fixmap_slot_addr, FIXMAP_HPET};
    // ← Fonction incomplète : aucun mapping effectué
}
```

Sur QEMU avec identity map, l'adresse HPET 0xFED00000 peut fonctionner par accident. Sur bare-metal, cette page n'est pas dans la physmap initiale → page fault ou accès UB.

## Correction

```rust
// kernel/src/arch/x86_64/acpi/hpet.rs

fn map_hpet_mmio_fixmap(hpet_phys_base: u64) -> Result<u64, HpetError> {
    use crate::arch::x86_64::paging::{map_4k_page, PageFlags};
    use crate::memory::core::layout::{fixmap_slot_virt, FixmapSlot};

    // Obtenir l'adresse virtuelle du slot fixmap réservé pour le HPET.
    let hpet_virt = fixmap_slot_virt(FixmapSlot::Hpet);

    // Flags MMIO : Uncacheable (UC) + No-Execute + Write (no cache).
    let flags = PageFlags::PRESENT
        | PageFlags::WRITABLE
        | PageFlags::NO_EXECUTE
        | PageFlags::NO_CACHE       // UC — obligatoire pour MMIO
        | PageFlags::WRITE_THROUGH; // WT si UC non disponible

    // Mapper la page physique HPET dans le fixmap.
    // SAFETY: hpet_phys_base est fourni par l'ACPI HPET table (validé par ACPI parser).
    unsafe {
        map_4k_page(
            hpet_virt,
            crate::memory::core::PhysAddr::new(hpet_phys_base),
            flags,
        ).map_err(|_| HpetError::MmioMapFailed)?;
    }

    // Invalider le TLB pour cette adresse.
    unsafe {
        core::arch::asm!(
            "invlpg [{0}]",
            in(reg) hpet_virt.as_u64(),
            options(nostack, preserves_flags)
        );
    }

    Ok(hpet_virt.as_u64())
}

// Ajouter FixmapSlot::Hpet dans le layout :
// kernel/src/memory/core/layout.rs
pub enum FixmapSlot {
    // ... slots existants ...
    Hpet,   // 1 page (4KB) pour le MMIO HPET
    // ...
}
```

## Correction

```rust
// servers/vfs_server/src/main.rs

async fn handle_umount(mount_point: &str) -> VfsResult<()> {
    // Étape 1 : bloquer les nouvelles opérations sur ce point de montage.
    let mount = MOUNT_TABLE.get(mount_point)
        .ok_or(VfsError::NotMounted)?;
    mount.set_unmounting();

    // Étape 2 : attendre la fin des opérations en cours (drain).
    mount.drain_pending_ops().await;

    // Étape 3 : flush du journal ExoFS (write-back de toutes les pages dirty).
    if let Some(exofs) = mount.exofs_instance() {
        exofs.flush_journal().await
            .map_err(|e| VfsError::FlushFailed(e))?;
        
        // Étape 4 : sync des métadonnées (superblock, bitmap d'allocation).
        exofs.sync_metadata().await
            .map_err(|e| VfsError::SyncFailed(e))?;
    }

    // Étape 5 : libérer les ressources du mount point.
    MOUNT_TABLE.remove(mount_point);

    Ok(())
}
```

# CORR-68 — wait_ch2_done timeout par itérations + SIPI unique (MAJ-07 + MAJ-09)

**Sévérité :** 🟠 MAJEURE  
**Fichiers :** `kernel/src/arch/x86_64/time/sources/pit.rs:192`, `kernel/src/exophoenix/stage0.rs:1129`

---

## Correction MAJ-07 — `wait_ch2_done()` basé sur TSC

```rust
// kernel/src/arch/x86_64/time/sources/pit.rs

/// Attend la fin du comptage PIT CH2 en vérifiant le bit OUT2.
/// Timeout : 15ms (le PIT CH2 est configuré pour ~10ms).
///
/// Utilise le TSC (via RDTSC) si disponible, sinon itérations.
/// Viole la règle CAL-WINDOW-01 si basé sur itérations → TSC obligatoire.
pub fn wait_ch2_done() -> bool {
    // Lire la fréquence TSC si déjà calibrée (boot tardif).
    // En early boot : utiliser une estimation conservatrice de 1 GHz.
    let tsc_per_ms: u64 = crate::arch::x86_64::time::ktime::boot_tsc_khz()
        .map(|khz| khz) // khz = ticks par ms
        .unwrap_or(1_000_000); // fallback 1 GHz = 1_000_000 ticks/ms

    let timeout_ticks = tsc_per_ms * 15; // 15ms timeout

    let start = unsafe { core::arch::x86_64::_rdtsc() };

    loop {
        // Lire le bit 5 du port 0x61 (OUT2 = PIT CH2 output).
        let val = unsafe { crate::arch::x86_64::io::inb(0x61) };
        if val & 0x20 != 0 {
            return true; // OUT2 = HIGH → PIT a fini
        }

        // Vérification timeout basé sur TSC (CAL-WINDOW-01).
        let elapsed = unsafe { core::arch::x86_64::_rdtsc() }.wrapping_sub(start);
        if elapsed >= timeout_ticks {
            return false; // Timeout
        }

        core::hint::spin_loop();
    }
}
```

```rust
pub fn send_sipi_once(core_slot: u8, entry_vector: u8) -> Result<(), SendSipiError> {
    let prior = SIPI_SENT.fetch_or(SIPI_SENT_BIT, Ordering::AcqRel);
    if (prior & SIPI_SENT_BIT) != 0 {
        return Err(SendSipiError::AlreadySent);
    }

    let apic_id = /* ... lookup apic_id depuis core_slot ... */;

    // Premier SIPI.
    ipi::send_startup_ipi(apic_id, entry_vector);

    // Délai 1ms entre les deux SIPIs (spec Intel MP §B.4).

    ipi::send_startup_ipi(apic_id, entry_vector);

/// Attend approximativement 1ms sans dépendre d'un timer externe.
/// Utilisé uniquement dans la séquence SIPI (contexte très early).
#[inline]
fn wait_1ms_approx() {
    // Utiliser ktime si le TSC est calibré.
    if let Some(khz) = crate::arch::x86_64::time::ktime::boot_tsc_khz() {
        let ticks = khz; // khz = ticks par ms
        let start = unsafe { core::arch::x86_64::_rdtsc() };
        while unsafe { core::arch::x86_64::_rdtsc() }.wrapping_sub(start) < ticks {
            core::hint::spin_loop();
        }
    } else {
        // Fallback : boucle calibrée empiriquement pour early boot (~1ms à ~1GHz).
        for _ in 0..1_000_000u32 {
            unsafe { core::arch::asm!("nop", options(nomem, nostack)); }
        }
    }
}
```

# CORR-69 — TSC offset signé stocké en u64 (MAJ-10)

**Sévérité :** 🟠 MAJEURE — Temps négatifs/overflow si AP en retard  
**Fichier :** `kernel/src/arch/x86_64/time/ktime.rs:288`

---

## Problème

```rust
static TSC_OFFSETS: [AtomicU64; MAX_CPUS] = ...; // u64 mais delta peut être négatif

// ktime.rs:100
let tsc_adjusted = tsc_now.wrapping_sub(
    super::percpu::tsc_offset(coreid as usize) // u64 — FAUX si négatif
);
// Si AP en retard : offset "négatif" stocké comme u64::MAX - |offset|
// wrapping_sub(u64::MAX - |offset|) = tsc_now + |offset| + 1 → explosion
```

## Correction

```rust
// kernel/src/arch/x86_64/time/ktime.rs

// REMPLACER :
// static TSC_OFFSETS: [AtomicU64; MAX_CPUS] = ...;
// PAR (représentation two's complement explicite) :

/// Offsets TSC per-CPU stockés en u64 (two's complement).
/// Un offset négatif (AP en retard sur le BSP) est stocké comme
/// l'interprétation u64 du i64 correspondant.
/// 
/// Convention :
///   - AP en avance sur BSP : offset positif (petit u64)
///   - AP en retard sur BSP : offset négatif (grand u64, > i64::MAX)
static TSC_OFFSETS: [AtomicU64; MAX_CPUS] = { /* same init */ };

/// Stocke l'offset TSC d'un CPU (signé, encodé en u64 two's complement).
pub unsafe fn store_tsc_offset(cpu_id: usize, offset_signed: i64) {
    if cpu_id >= MAX_CPUS { return; }
    // Encoder i64 en u64 two's complement (transmute sûr).
    let encoded = offset_signed as u64;
    TSC_OFFSETS[cpu_id].store(encoded, Ordering::Release);
    TSC_OFFSET_VALID[cpu_id].store(true, Ordering::Release);
}

/// Retourne l'offset TSC signé d'un CPU.
pub fn tsc_offset_signed(cpu_id: usize) -> i64 {
    if cpu_id >= MAX_CPUS { return 0; }
    if TSC_OFFSET_VALID[cpu_id].load(Ordering::Relaxed) {
        TSC_OFFSETS[cpu_id].load(Ordering::Relaxed) as i64 // two's complement
    } else {
        0
    }
}

// ktime.rs:100 — CORRIGER le calcul :
let tsc_adjusted = (tsc_now as i64)
    .wrapping_sub(tsc_offset_signed(coreid as usize)) as u64;
// wrapping_sub sur i64 gère correctement les offsets négatifs.
```

```rust

/// Préférer tsc_offset_signed() pour les calculs.
#[deprecated(note = "Use tsc_offset_signed() to avoid sign confusion")]
pub fn tsc_offset(cpu_id: usize) -> u64 {
```

# CORR-70 — SPSC ring collision par modulo (MAJ-12)

**Sévérité :** 🟠 MAJEURE — Corruption IPC inter-processus  
**Fichier :** `kernel/src/ipc/ring/spsc.rs:298`

---

## Problème

```rust
const MAX_SPSC_RINGS: usize = 256;
fn ring_for(channel_id: u64) -> &'static SpscRing {
    let idx = array_index_nospec((channel_id as usize) % MAX_SPSC_RINGS, MAX_SPSC_RINGS);
    &SPSC_RINGS[idx]
}
// Canal 0 et canal 256 → même ring → messages mélangés
```

## Correction

```rust
// kernel/src/ipc/ring/spsc.rs

// Approche 1 : Table de mapping channel_id → ring index (hashtable ou bitmap)

/// Table de mapping channel_id → ring index.
/// Chaque channel_id actif est alloué un ring index unique.
static CHANNEL_MAP: spin::Mutex<ChannelMap> = spin::Mutex::new(ChannelMap::new());

struct ChannelMap {
    /// Map channel_id → ring index.
    entries: [Option<(u64, usize)>; MAX_SPSC_RINGS],
    /// Prochain ring index libre.
    next_free: usize,
}

impl ChannelMap {
    const fn new() -> Self {
        Self {
            entries: [None; MAX_SPSC_RINGS],
            next_free: 0,
        }
    }

    /// Alloue un ring index pour un channel_id.
    /// Retourne None si tous les rings sont utilisés.
    fn alloc(&mut self, channel_id: u64) -> Option<usize> {
        // Vérifier si déjà alloué.
        for (cid, idx) in self.entries.iter().flatten() {
            if *cid == channel_id { return Some(*idx); }
        }

        // Trouver un slot libre.
        for i in 0..MAX_SPSC_RINGS {
            let slot = (self.next_free + i) % MAX_SPSC_RINGS;
            if self.entries[slot].is_none() {
                self.entries[slot] = Some((channel_id, slot));
                self.next_free = (slot + 1) % MAX_SPSC_RINGS;
                return Some(slot);
            }
        }
        None // Tous les rings sont utilisés
    }

    fn get(&self, channel_id: u64) -> Option<usize> {
        self.entries.iter().flatten()
            .find(|(cid, _)| *cid == channel_id)
            .map(|(_, idx)| *idx)
    }

    fn free(&mut self, channel_id: u64) {
        for slot in self.entries.iter_mut() {
            if let Some((cid, _)) = slot {
                if *cid == channel_id { *slot = None; return; }
            }
        }
    }
}

fn ring_for(channel_id: u64) -> Option<&'static SpscRing> {
    let idx = CHANNEL_MAP.lock().get(channel_id)?;
    Some(&SPSC_RINGS[idx])
}

/// Alloue un ring pour un nouveau canal.
/// Doit être appelé lors de la création du canal IPC.
pub fn alloc_ring_for_channel(channel_id: u64) -> Result<(), IpcError> {
    CHANNEL_MAP.lock()
        .alloc(channel_id)
        .map(|_| ())
        .ok_or(IpcError::NoRingsAvailable)
}

/// Libère le ring d'un canal fermé.
pub fn free_ring_for_channel(channel_id: u64) {
    CHANNEL_MAP.lock().free(channel_id);
}
```

---

**Note CORR-70 :** L'approche hashtable est plus robuste mais nécessite de modifier le protocole d'ouverture de canal. En attendant, une solution **minimale** acceptable est de limiter `MAX_SPSC_RINGS` à 256 et d'**imposer que `channel_id < 256`** via une assertion dans `alloc_ring_for_channel()`.
